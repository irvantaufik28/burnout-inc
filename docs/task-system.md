# Task and Action System

## System Identity
Controls what the player is currently doing. In Phase 2, this has shifted from manual project work to Management and Recovery Actions.

## Dual Scale Integration
Tasks progress in **Project Time** (fast), allowing for quick recovery and pacing decisions. However, the benefits of these tasks (e.g., energy gained from a Nap) are balanced against the slower **Human Time** decay scale.

## Action Types
- **Take Nap (4h):** High Energy recovery.
- **Take Quick Break (2h):** High Focus recovery.
- **Stimulants:** Instant benefits with delayed aftermaths.
- **Deep Sleep (6h):** Max recovery for both Energy and Focus.

## Execution Rules
- Players select recovery actions to keep the Auto Work System running efficiently.
- Task completion triggers logic based on the action type.
