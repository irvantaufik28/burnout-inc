# Unexpected Expense System

## System Identity
Life is unpredictable. This system introduces rare, grounded disruptions that require immediate financial and strategic decisions.

## Transition to Pending Actions
Some unexpected expenses (specifically Device issues) can be "Ignored." Doing so saves immediate capital but transitions the event into a persistent **Pending Maintenance** issue with Focus penalties.

## Event Triggers
- Rarity: Very rare.
- Cooldown: Minimum 5 days for general events, 10 days for device-specific damage.
- Chance: Randomized low probability per game hour.

## Decision Architecture
Every event presents at least two options:
- Option A (Expensive/Safe): Usually restores vitals or improves technical expertise.
- Option B (Cheap/Risky): Saves money but introduces technical debt or persistent hardware damage.
