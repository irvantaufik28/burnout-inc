# Pending Action System

## System Identity
The Pending Action system manages long-term obligations and deferred tasks. It ensures that "saying no" to an immediate problem has lasting consequences.

## Architecture
Pending actions are persistent states that stay in the store until resolved. They differ from Project Conditions because they do not expire on their own.

## Current Categories
- **Device Maintenance:** Repairs for hardware damage (Keyboard, Laptop, etc.).
- **Unpaid Bills (Future):** Deferred financial settlements.
- **Administrative Debt (Future):** Taxes, registration, and licenses.

## UI Integration
Pending actions are displayed in the **Action Panel** under a dedicated "Pending Maintenance" section, ensuring they compete with daily tasks and contracts for the player's attention and capital.
