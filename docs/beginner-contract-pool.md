# Beginner Contract Pool (LV 1–10)

## Distribution Strategy
To ensure diversity, contracts are generated using a weighted distribution:
- **Frontend:** 30% (Visible and easy to understand).
- **Backend:** 25% (Foundation of architecture).
- **AI:** 15% (Modern hype and experimentation).
- **DevOps:** 15% (Stability and operations).
- **Mobile:** 15% (Device-specific UX).

## Project Metadata
Beginner projects follow a "low requirement" rule (usually Level 1–3) to ensure accessibility.

### Examples by Domain
- **Frontend:** Portfolio Landing, UI Redesign, Dashboard Styling.
- **Backend:** Basic REST API, Auth Setup, CRUD System.
- **AI:** Prompt Template Tool, Chat Widget, Content Generator MVP.
- **DevOps:** VPS Deployment, Docker Setup, CI/CD Fix.
- **Mobile:** Expo Prototype, Push Notification Setup, Mobile UI Fix.

## Domain Chaos (Beginner)
Events are tailored to teach domain identity:
- **Frontend Chaos:** "Client wants a different blue".
- **Backend Chaos:** "Authentication token issues".
- **AI Chaos:** "Model hallucinating features".
- **DevOps Chaos:** "Container failed to restart".
